Activities
O que são?
A classe de Activities é um dos componentes básicos de uma aplicativo Android. Paralelamente à outros apps que são lançados com um método main(), as aplicações Android iniciam seu funcionamento em uma instância Activity e a manipula por meio das callbacks.
Conceito
Como a navegação em aplicativos Mobile podem iniciar em lugares diferentes, ex: Whatsapp - (Lista de Conversas, Conversas), foi necessário a criação de algo que auxiliasse esse paradigma. Assim, ao implementar uma Activity como uma subclasse da classe Activity, a atividade serve como ponto inicial da interação do usuário com o aplicativo e pode iniciar também outras atividades, inclusive, de outros aplicativos. Embora as Activities trabalhem juntas, elas normalmente são independentes entre si, ou seja, tem ligações mínimas uma com as outras.

Normalmente, uma atividade implementa uma tela ou uma janela em uma aplicação, e como a maioria dos aplicativos possuem várias telas, logo eles possuem várias atividades. É comum também que um aplicativo tenha uma Activity principal, a qual se designa como a primeira tela a ser exibida para o usuário ao iniciar o App.
Criando uma Activity
Para criar uma atividade, é necessário abrir o arquivo de manifesto e adicionar um elemento <activity> como filho do elemento <application>, Note que o nome é obrigatório. Como em: <manifest ... > <application ... > <activity android:name=".ExampleActivity" /> ... </application ... > ... </manifest >
Manipulando uma Activity
Por passarem por vários estados enquanto funcionam, as Activities são manipuladas por meio de métodos callbacks, na qual cada callback é chamada quando a Activity atinge certo estado. São exemplos de Callbacks:

onCreate(): Callback obrigatória para o sistema e é iniciada quando o sistema cria a Activity. É esta callback que inicia os componentes essenciais da Activity, sendo também, o lugar onde é definido o layout da interface.

onStart(): Callback iniciada quando a OnCreate() termina e a interface prepara-se para tornar-se visível e interativa ao usuário.

onPause(): Callback iniciada quando a Activity torna-se segundo plano para o usuário. Uma atividade pausada pode continuar atualizando sua interface se necessário.

onResume(): Callback iniciada assim que o usuário começa a interagir com a atividade, a maior parte de uma Activity é implementada neste método.

onStop(): Callback iniciada quando a atividade deixa de ser visível ao usuário e torna-se interrompida.

onRestart(): Callback iniciada quando a atividade interrompida está para ser reiniciada novamente.

onDestroy(): Callback iniciada pouco antes da atividade ser destruída. Esta é a última callback recebida a partir da Activity.
