Content Providers
Conceito
Os Content Providers ou Provedores de Conteúdo são os responsáveis por gerenciar o acesso a uma central de dados e funcionam de maneira parecida com um banco de dados relacional. Geralmente, o Provedor fornece sua própria UI que pode ser utilizada para manipular os dados. Os Provedores são majoritariamente destinados ao seu uso por outros aplicativos, que para se conectarem, fazem uso de um novo objeto conhecido como Cliente do Provedor - essa relação cria uma interface consistente para a troca de dados.

Costumam ser utilizados quando você pretende acessar um provedor de conteúdo existente ou criar o próprio provedor para fornecer acesso para outros aplicativos

Acessar o Provedor
Para acessar um Provedor é necessário utilizar um objeto do tipo do ContentResolver no Context do aplicativo para tornar-se cliente do Provedor. O ContentResolver comunica com o ContentProvider que performa a ação pedida e retorna os resultados. O ContentResolver pode realizar todas as ações de um CRUD.
URI's de Conteúdo
Um URI de Conteúdo é um URI que identifica os dados de um Provedor, eles contêm o nome simbólico do Provedor e do identificador da tabela. Ao chamar um método para acessar uma tabela do Provedor, o URI de Conteúdo da tabela é um dos parâmetros necessários.
Consultas
Construção: Para recuperar dados de um Provedor, é primeiramente necessário criar uma consulta e definir algumas váriaveis para acessar o Provedor desejado.

Exibição: Para exibir os dados, é necessário usar o método ContentResolver.query() que retorna um objeto Cursor contendo o conteúdo do Provedor para as colunas reqeridas na consulta. O Cursor possui métodos que possibilitam iterar, contar resultados, extrair dados, etc. É possível também usar o resultado de um Provedor para pesquisar em outro.
CRUD
Para realizar o CRUD no Provedor, utiliza-se o objeto ContentResolver e chama-se o método correspondente.

Inserção: O método ContentResolver.insert() é utilizado. O método insere conteúdo no provedor e retorna o conteúdo dessa linha.

Atualização: ContentResolver.update() é utilizado. É necessário passar um parâmetro do tipo ContentValues, só é necessário adicionar valores no ContentValues para os campos que você pretende atualizar.

Exclusão: ContentResolver.delete() é utilizado. É necessário especificar a seleção, informando o user que será deletado.
