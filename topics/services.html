Services
O que são?
Um Service é um componente que pode efetuar operações longas e que não fornece uma interface do usuário. Outros componentes podem iniciar e vincular-se a um serviço para interagir. Um serviço pode reproduzir música, executar E/S de arquivos imagens e interagir com um provedor de conteúdo a partir do segundo plano.
Conceito

Primeiro Plano: O serviço realiza uma tarefa que é perceptível para o usuário. Reproduzir uma faixa de áudio é uma tarefa em 1° plano e continuam sendo executados mesmo se o usuário não estiver interagindo com o sistema.

Segundo Plano: O serviço realiza uma tarefa que é inperceptível para o usuário. Compactamento de armazenamento é normalmente, uma tarefa em 2° plano.

Vinculado: Um serviço vinculado é um serviço que permanece em execução enquanto o Serviço vinculado a ele estiver funcionando. Um serviço pode vincular o outro utilizando-se de bindService() e Serviços vinculados oferecem uma interface cliente-servidor que permite que os componentes interajam.
Declarando e Iniciando um Serviço
Para declarar um serviço, é necessário usar o manifesto e adicionar um elemento <service> como filho do elemento <application>, assim como nas atividades, apenas o nome é obrigatório. Exemplo: <manifest ... > ... <application ... > <service android:name=".ExampleService" /> ... </application> </manifest> Para iniciar um serviço, é necessário utilizar o comando startService() em algum outro componente, resultando no método onStartCommand() para o serviço. Um serviço iniciado funciona por tempo indeterminado até que seja concluído ou que seja parado por outro componente via stopService().
Manipulando um Service
O ciclo de vida de um Serviço pode seguir dois modos - vinculado ou iniciado, sendo que o vinculado é criado quando um componente chama bindService() e o iniciado é criado quando um componente chama startService().

Assim como as Activities, um Service possui callbacks, que são:

onCreate(): O serviço está sendo criado.

onStartCommand(): Um serviço iniciado está iniciando.

onBind(): Um cliente está se vinculando ao serviço.

onUnbind(): Todos os clientes se desvincularam do serviço.

onRebind(): Um cliente está se revinculando ao serviço (depois de onUbind().

onDestroy(): O serviço foi destruído e não pode mais ser utilizado.
